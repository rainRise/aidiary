## 📋 规则概览

本项目使用 Qoder AI 辅助开发，为确保开发质量，制定了以下规则体系。

### 规则执行流程

```
规则 1: 需求理解 (xuqiu_lijie.mdc)
  ↓ 确认理解正确
规则 2: 规划思考 (guihua.mdc)
  ↓ 方案制定完成
规则 3: 开发执行 (kaifa.mdc)
  ↓ 代码开发完成
规则 4: 开发日志 (rizhi.mdc)
  ↓ 持续记录
规则 5: 质量控制 (zhiliang.mdc)
  ↓
规则 6: 问题处理 (wenti.mdc) ← 按需触发
```

## 🎯 核心原则

### 必须遵守

1. **先理解，后规划，再开发**
   - ❌ 禁止接到需求直接写代码
   - ✅ 必须先确认理解用户需求
   - ✅ 必须制定详细实施计划
   - ✅ 必须获得用户确认后再开发

2. **小步快跑，持续验证**
   - ✅ 每个功能拆分为小步骤 (≤ 800 行/步)
   - ✅ 每完成一个步骤立即验证
   - ✅ 发现问题立即修复

3. **文档先行**
   - ✅ 每次开发后更新相关文档
   - ✅ 记录开发日志
   - ✅ 更新项目状态

4. **代码质量**
   - ✅ 遵循项目代码规范
   - ✅ 完善的错误处理
   - ✅ 类型安全 (TypeScript + Pydantic)
   - ✅ 添加必要注释

### 严格禁止

1. ❌ 禁止删除用户现有代码
2. ❌ 禁止修改配置文件 (.env) 除非用户明确要求
3. ❌ 禁止提交不完整的代码
4. ❌ 禁止跳过测试和验证
5. ❌ 禁止使用过时的 API 或方法
