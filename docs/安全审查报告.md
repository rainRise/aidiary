# 映记安全审查报告

**审查日期**: 2026-04-03
**项目**: 映记 (Yinji) 智能日记应用
**技术栈**: FastAPI + React + TypeScript + PostgreSQL + Qdrant

---

## 执行摘要

本报告对映记项目进行了全面的安全审查。发现 **3 个严重级别问题、4 个高级别问题、5 个中等级别问题、3 个低级别问题**。严重问题主要包括暴露的凭证、未受保护的邮件接口以及 localStorage 中的 JWT 存储，需要立即处理。

### 3. JWT Token 存储在 localStorage

**涉及文件**:
- `frontend/src/store/authStore.ts` 第 42、62、103、108、130 行
- `frontend/src/services/api.ts` 第 17 行
- `frontend/src/services/assistant.service.ts` 第 73 行

```typescript
localStorage.setItem('access_token', response.access_token)
// ...
const token = localStorage.getItem('access_token')
```

**影响**:
- 任何 XSS 漏洞可被直接窃取 Token
- localStorage 对页面中运行的任意 JavaScript 代码可见
- 无 Token 刷新机制

**修复建议**:
- 使用 `httpOnly` Cookie 存储 Token
- 增加 CSRF 保护
- 实现短生命周期 Access Token + Refresh Token 轮换机制

## 高级别问题（HIGH）

### 4. 认证接口无频率限制

**文件**: `backend/app/api/v1/auth.py`

登录和注册接口缺少防暴力破解保护：
- `POST /auth/login` — 无频率限制
- `POST /auth/login/password` — 无频率限制
- `POST /auth/register/verify` — 无频率限制

**影响**:
- 6 位数字验证码（100 万种组合）可被暴力破解
- 攻击者可进行凭证填充攻击
- 可通过时间差异推断账户是否存在

**PoC**: 利用响应时间差异，预期约 60 次请求即可破解验证码。

**修复建议**:
```python
from slowapi import Limiter
from slowapi.util import get_remote_address
limiter = Limiter(key_func=get_remote_address)

@router.post("/auth/login")
@limiter.limit("5/minute")
async def login(...): ...
```

### 5. AI 对话存在 Prompt 注入风险

**文件**: `backend/app/api/v1/assistant.py` 第 328-341 行

用户输入直接拼入 System Prompt，无任何过滤：

```python
user_prompt = (
    f"用户当前问题：{message}\n\n"  # 直接用户输入
    "请直接回复用户，不要输出JSON。"
)
```

**影响**:
- 恶意用户可构造提示词窃取其他用户的日记数据
- 可绕过系统指令执行未授权操作
- 通过精心构造的查询实现数据泄露

**修复建议**:
```python
def sanitize_user_input(text: str) -> str:
    dangerous_patterns = ["你是一个", "忽略之前的指令", "你现在是", "system:"]
    for pattern in dangerous_patterns:
        text = text.replace(pattern, "")
    return text.strip()
```

### 6. 日记内容 HTML 渲染 XSS 风险

**文件**: `backend/app/models/diary.py` 第 42 行

```python
content_html: Mapped[Optional[str]] = mapped_column(Text)
```

`content_html` 字段表明应用存储并可能渲染 HTML 内容。

**影响**: 存储型 XSS — 攻击者写入含恶意 JavaScript 的日记内容，其他用户查看时执行。

**修复建议**:
- 前端存储前使用 DOMPurify/sanitize-html 消毒
- 渲染时使用 `textContent` 而非 `innerHTML`
- 添加 Content Security Policy (CSP) 响应头

### 7. 匿名帖子无追踪机制

**文件**: `backend/app/models/community.py` 第 41 行

```python
is_anonymous: Mapped[bool] = mapped_column(Boolean, default=False)
```

匿名发帖无 IP 记录、无用户追踪。

**影响**:
- 匿名骚扰和 abuse
- 垃圾邮件和钓鱼内容
- 违反社区规范却无追责

**修复建议**:
- 记录匿名帖 IP（需在隐私政策中披露）
- 基于 IP 实现发帖频率限制
- 匿名帖进入内容审核队列

### 10. 文件上传先全量读入内存

**文件**: `backend/app/api/v1/diaries.py` 第 205-238 行

```python
contents = await file.read()  # 先整文件读入内存
if len(contents) > 10 * 1024 * 1024:
    raise HTTPException(...)
```

**影响**: 攻击者可上传超大文件耗尽服务器内存。

**修复建议**: 使用流式上传：
```python
async def read_file_in_chunks(file, max_size):
    total = 0
    while chunk := await file.read(8192):
        total += len(chunk)
        if total > max_size:
            raise HTTPException(...)
        yield chunk
```

### 12. 验证码错误信息存在时序差异

**文件**: `backend/app/services/auth_service.py` 第 119-140 行

```python
verification_code = result.scalar_one_or_none()
if not verification_code:
    return False, "验证码错误"  # 时间 T1
if verification_code.expires_at < datetime.utcnow():
    return False, "验证码已过期"  # 时间 T2
return False, "验证码错误"  # 时间 T3
```

"不存在"、"已过期"、"错误" 三种错误响应时间不同，攻击者可通过时序分析判断账户是否存在。

**修复建议**: 使用恒定时间比较：
```python
import hmac
def constant_time_compare(val1: str, val2: str) -> bool:
    return hmac.compare_digest(val1.encode(), val2.encode())
```

## 低级别问题（LOW）

### 13. 密码最小长度仅 6 位

**文件**: `backend/app/schemas/auth.py` 第 35、48、95 行

```python
password: str = Field(..., min_length=6, max_length=50, description="密码")
```

**修复建议**: 改为 `min_length=8`，并考虑要求混合大小写、数字和特殊字符。

---

### 14. Token 过期时间 7 天过长

**文件**: `backend/app/core/config.py` 第 34-36 行

```python
access_token_expire_minutes: int = Field(default=10080)  # 7天
```

对于含敏感个人数据的心理日记应用，7 天无刷新机制风险过高。

**修复建议**: Access Token 15-60 分钟有效，配合 Refresh Token 轮换。

---

### 15. 缺少安全响应头

**文件**: `backend/main.py`

未配置安全响应头（CSP、X-Frame-Options、X-Content-Type-Options 等）。

**修复建议**:
```python
@app.middleware
async def add_security_headers(request, call_next):
    response = await call_next(request)
    response.headers["X-Content-Type-Options"] = "nosniff"
    response.headers["X-Frame-Options"] = "DENY"
    response.headers["Content-Security-Policy"] = "default-src 'self'"
    return response
```

- ✅ SQL 使用参数化查询（SQLAlchemy ORM）
- ✅ 密码使用 bcrypt 哈希

- ✅ 邮箱使用 `EmailStr` 验证
- ✅ 文件上传检查 `content_type`
- ✅ 操作验证所有权（`user_id` 检查）
- ✅ 无 `eval()` 或 `exec()` 危险动态代码执行
- ✅ 前端未发现 `dangerouslySetInnerHTML`
