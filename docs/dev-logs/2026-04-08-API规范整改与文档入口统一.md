### 3. API 文档入口统一
- 改动文件：`backend/main.py`

## 二、遇到的问题与解决方案

### 问题 1: 直接统一成功响应会破坏前端
**原因**: 现有前端大量依赖原始 response_model（非 envelope）。  
**解决**: 本次先统一错误响应并兼容 `detail`，成功响应治理放入下一阶段（建议 v2 路由渐进迁移）。
