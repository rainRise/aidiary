# 2026-04-05 开发日志：情绪特征向量多词典融合升级

## 背景

现有 `emotion_feature_service.py` 使用内置 VAD 词典，解释性强但词汇覆盖有限。  
为提升中文情绪词覆盖率与鲁棒性，本次实现“外部词典可插拔融合”能力，优先兼容 NTUSD 类极性词典。

## 本次改动

## 1. 核心服务升级

文件：`backend/app/services/emotion_feature_service.py`

新增能力：

1. 初始化时加载外部词典（可开关）。  
2. 支持正/负词表文件解析（每行一个词或“词+分数”格式）。  
3. 将外部极性词映射为 VAD 原型值，并按权重与内置词典融合。  
4. 统一让 `extract` / `explain` 都基于融合后的 `self.vad_lexicon` 执行。

关键参数（环境变量）：

1. `EMOTION_EXT_LEXICON_ENABLED`（默认 `true`）  
2. `EMOTION_EXT_LEXICON_WEIGHT`（默认 `0.65`）  
3. `EMOTION_EXT_LEXICON_POS_PATH`  
4. `EMOTION_EXT_LEXICON_NEG_PATH`

设计原则：

1. 外部词典缺失时自动跳过，不影响启动。  
2. 已有词条做加权融合，避免“硬覆盖”导致风格突变。  
3. 所有分值做边界裁剪，保证 VAD 值域合法。

## 2. 词典目录规范

文件：`backend/app/data/lexicons/README.md`

内容包括：

1. 默认文件名约定（`ntusd_positive_simplified.txt` / `ntusd_negative_simplified.txt`）。  
2. 支持的词典格式说明。  
3. 环境变量配置方式。

## 验证与预期

1. 无外部词典文件时：服务行为与原先一致（仅日志提示跳过）。  
2. 提供词典文件后：情绪命中词覆盖率上升，`explain.matched_emotions` 更丰富。  
3. 后续建议：基于标注集做 A/B 离线评估，继续调优外部词典权重。

## 后续计划

1. 增加“词典融合命中率”监控指标。  
2. 结合真实用户语料做词典增量清洗（去噪词/歧义词）。  
3. 在不牺牲可解释性的前提下，探索轻量模型校准层。
