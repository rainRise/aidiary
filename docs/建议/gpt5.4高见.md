第一步：把当前方案从“能用”升级到“更稳”
1. 检索打分不要只看 BM25

加几个附加权重：

时间权重：越近越重要
重要性权重：用户手动标记 or 模型抽取的重要事件
情绪强度权重：情绪越强的片段分更高
重复出现权重：同类主题反复出现要加权
人物命中权重：重要人物相关内容加权

也就是变成：

final_score = bm25_score + recency + importance + emotion_intensity + repetition

这样你就不是纯词法检索了，
而是“更像综合分析的召回排序”。

2. 证据去重

很重要。

你要防止：

同一篇日记多个 chunk 重复入选
多个 query 命中同一主题的重复证据

建议：

同一 entry 限制入选 chunk 数
对相似 chunk 做聚类 / 去重
每个分析维度保留 2~4 条代表性证据，而不是越多越好
3. 不只检索 raw chunk，也检索“日摘要”

这是非常值得加的一步。

你现在可以保留两层证据：

原始片段
每篇日记的简摘要

这样模型既能看见细节，又不容易迷失。

第二步：加“长期记忆层”，但不用上大图谱
你最应该补的不是 Neo4j

而是这三样：

1）每日日记摘要

每篇日记写完后，后台生成：

事件摘要
主情绪
关键人物
是否有未解决问题
是否有高强度冲突
2）每周 / 每月摘要

自动生成：

本周情绪走势
关键冲突
反复主题
支持源 / 压力源
3）长期画像

沉淀：

常见压力触发器
典型应对姿态
高重复关系模式
高频价值冲突

有了这三层以后，“综合分析”才会从：
最近总结
变成
近期状态 + 长期模式对照

第三步：再加轻量语义检索，而不是直接全量 GraphRAG

这一阶段再考虑：

pgvector
或 Chroma
或混合检索（BM25 + 向量）

我建议未来路线是：

Hybrid Retrieval
BM25 负责字面高精度命中
向量负责语义补召回
最后统一 rerank

这个比“纯向量”更稳。
尤其是日记这种文本，关键词、人物名、事件词、时间线都很重要。
