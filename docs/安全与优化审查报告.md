## 2. 安全审查发现 (Security Findings)

### 2.1 严重级别 (Critical Severity)

#### C1: 生产环境凭证暴露于 `.env` 文件

**风险等级**: Critical  
**影响范围**: 完整服务接管  
**相关文件**: `backend/.env`

**问题描述**:
`.env` 文件包含以下敏感凭证：
- DeepSeek API Key
- Qdrant 向量数据库凭证
- SMTP 邮箱密码
- JWT Secret Key

**攻击场景**:
攻击者获取代码库访问权限后可直接读取所有生产凭证，完全控制 AI 服务、数据库和邮件系统。

**修复建议**:
```bash
# 1. 立即轮换所有凭证
# 2. 使用 secrets manager (如 AWS Secrets Manager, Azure Key Vault)
# 3. 将 .env 添加到 .gitignore 并清理 Git 历史
```

```python
# 使用环境变量或 secrets manager
import os
from azure.identity import DefaultAzureCredential
from azure.keyvault.secrets import SecretClient

credential = DefaultAzureCredential()
client = SecretClient(vault_url="https://yinji-vault.vault.azure.net/", credential=credential)
deepseek_api_key = client.get_secret("deepseek-api-key").value
```

#### C2: 未受保护的邮件中继端点

**风险等级**: Critical  
**影响范围**: 邮件滥用、垃圾邮件发送、服务被列入黑名单  
**相关文件**: `backend/app/api/v1/auth.py`

**问题描述**:
`GET /test-email` 端点无需认证即可调用，可向任意邮箱地址发送邮件。

**漏洞代码**:
```python
@router.get("/test-email")
async def test_email(email: str):
    # 无认证检查，可被滥用
    await send_email(email, "Test", "Test content")
```

**修复建议**:
```python
from app.core.deps import get_current_admin_user

@router.get("/test-email")
async def test_email(
    email: str,
    current_user: User = Depends(get_current_admin_user)  # 仅管理员可访问
):
    await send_email(email, "Test", "Test content")
```

**替代方案**: 生产环境完全移除该端点。

#### C3: 不安全的 Cookie 配置

**风险等级**: Critical  
**影响范围**: MITM 攻击、Token 窃取、会话劫持  
**相关文件**: `backend/app/api/v1/auth.py`

**问题描述**:
`access_token` 和 `refresh_token` Cookie 设置了 `secure=False`，允许通过 HTTP 传输。

**漏洞代码**:
```python
response.set_cookie(
    key="access_token",
    value=access_token,
    secure=False,  # 危险：允许 HTTP 传输
    httponly=True,
    samesite="lax"
)
```

**修复建议**:
```python
import os

is_production = os.getenv("ENVIRONMENT") == "production"

response.set_cookie(
    key="access_token",
    value=access_token,
    secure=is_production,  # 生产环境强制 HTTPS
    httponly=True,
    samesite="strict" if is_production else "lax"
)
```

#### H2: AI 助手 Prompt Injection 风险

**风险等级**: High  
**影响范围**: LLM 被操控、信息泄露、不当内容生成  
**相关文件**: `backend/app/api/v1/assistant.py`

**问题描述**:
用户输入直接插 LLM Prompt，无过滤或输出验证。

**漏洞代码**:
```python
prompt = f"""
用户问题: {user_input}  # 直接插值，无过滤
请回答用户问题。
"""
```

**修复建议**:
```python
import re
from typing import Optional

def sanitize_input(text: str) -> str:
    # 移除潜在的 prompt injection 标记
    dangerous_patterns = [
        r'ignore previous instructions',
        r'system prompt',
        r'you are now',
        r'<script',
        r'javascript:',
    ]
    for pattern in dangerous_patterns:
        text = re.sub(pattern, '[REMOVED]', text, flags=re.IGNORECASE)
    return text[:1000]  # 限制长度

def validate_output(output: str) -> Optional[str]:
    # 检查输出是否包含敏感信息
    if any(keyword in output.lower() for keyword in ['password', 'secret', 'key', 'token']):
        return None
    return output
```

#### H5: 验证码时序攻击

**风险等级**: High  
**影响范围**: 验证码枚举  
**相关文件**: `backend/app/services/auth_service.py`

**问题描述**:
使用普通字符串比较验证验证码，存在时序攻击风险。

**漏洞代码**:
```python
def verify_code(self, code: str, stored_code: str) -> bool:
    return code == stored_code  # 时序攻击风险
```

**修复建议**:
```python
import hmac
import time

def verify_code(self, code: str, stored_code: str) -> bool:
    # 恒定时间比较
    is_valid = hmac.compare_digest(code.encode(), stored_code.encode())
    
    # 添加随机延迟防止时序分析
    time.sleep(random.uniform(0.1, 0.3))
    return is_valid
```

```typescript
      secure: true  // 验证 SSL 证书
```

```python
from starlette.middleware.base import BaseHTTPMiddleware

class SecurityHeadersMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        response = await call_next(request)
        response.headers["X-Content-Type-Options"] = "nosniff"
        response.headers["X-Frame-Options"] = "DENY"
        response.headers["X-XSS-Protection"] = "1; mode=block"
        response.headers["Strict-Transport-Security"] = "max-age=31536000; includeSubDomains"
```

## 3. 优化建议 (Optimization Findings)

### 3.1 高影响级别 (High Impact)

#### OH1: 社区服务 N+1 查询问题

**影响**: 95% 查询性能提升潜力  
**相关文件**: `backend/app/services/community_service.py`

**问题描述**:
获取帖子列表时，每个帖子触发额外查询获取作者、评论、点赞，产生 1+20×3=61 次查询。

**修复建议**:
```python
from sqlalchemy.orm import selectinload
from sqlalchemy import select

async def get_posts_with_eager_loading(db: AsyncSession, skip: int = 0, limit: int = 20):
    stmt = (
        select(Post)
        .options(
            selectinload(Post.author),
            selectinload(Post.comments).selectinload(Comment.author),
            selectinload(Post.likes)
        )
        .offset(skip)
        .limit(limit)
    )
    result = await db.execute(stmt)
    return result.scalars().all()  # 仅需 1 次查询
```

#### OH3: 冗余的 LLM API 调用

**影响**: 50-70% 成本节省  
**相关文件**: `backend/app/services/rag_service.py`

**问题描述**:
时间线优化被多次调用，相同输入重复请求 LLM。

**修复建议**:
```python
from functools import lru_cache
import hashlib

class TimelineService:
    def __init__(self):
        self._cache = {}
    
    async def refine_timeline(self, content: str) -> str:
        cache_key = hashlib.md5(content.encode()).hexdigest()
        
        if cache_key in self._cache:
            return self._cache[cache_key]
        
        result = await self._call_llm(content)
        self._cache[cache_key] = result
        return result
```
