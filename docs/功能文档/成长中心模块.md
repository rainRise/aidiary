# 成长中心

> 用日记画出你的情绪地图，看见自己的变化。

---

## 模块定位

成长中心是映记的**核心分析页面**，把你过去一段时间的日记浓缩成可视化的图表。
不需要自己翻日记——打开成长中心，一眼看到这段日子过得怎么样。

路由：`/growth`（旧路径 `/timeline` 会自动 302 重定向过来）

前端组件：`pages/timeline/Timeline.tsx`

---

## 页面结构

从上到下依次排列：

```
页头：← 返回  |  成长中心  |  写日记 →
├── 心情日历（MoodCalendar）
├── 能量值趋势图（EnergyTrend）
├── 萨提亚关键词卡片（SatirKeywords）
├── 人际关系图谱（占位，后续迭代）
└── 当日详情（DayDetail）
```

背景色为米白 `#FAF9F7`，整体风格与首页保持一致。

## 数据来源 API

成长中心依赖两个后端端点获取数据：

| 端点 | 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|------|
| `/api/v1/diaries/timeline/terrain` | GET | `days`（默认 30） | `TerrainResponse` | 返回每日聚合数据（能量/愉悦/密度/事件列表）+ 洞察（峰谷/趋势） |
| `/api/v1/diaries/growth/daily-insight` | GET | `target_date`（YYYY-MM-DD） | `GrowthDailyInsight` | 返回某日的 AI 生成洞察摘要，首次调用时后端生成并缓存 |

### TerrainResponse 结构

```ts
interface TerrainResponse {
  points: TerrainPoint[]      // 每日数据点
  insights: TerrainInsights   // 峰谷检测 + 趋势判断
  meta: TerrainMeta           // 元信息（范围、均值等）
}

interface TerrainPoint {
  date: string                // YYYY-MM-DD
  energy: number | null       // 能量值 1~10
  valence: number | null      // 愉悦度 -1.0 ~ 1.0
  density: number             // 事件密度（事件数或日记数兜底）
  events: TerrainEvent[]      // 当日事件列表
}
```

### 后端 terrain_service 处理逻辑

1. 查询时间窗口内的所有 `TimelineEvent` + `Diary` 记录
2. 按日聚合：能量取事件均值（缺失时用日记 `importance_score` 估算），愉悦度通过 `EMOTION_VALENCE` 映射表从情绪标签转换，密度取事件数（无事件时回退为日记篇数）
3. 峰谷检测 `detect_peaks_and_valleys()`：在能量序列上用滑窗找局部极值
4. 趋势判断 `detect_trend()`：判断能量整体走向（ascending / descending / stable）
5. 情绪愉悦映射覆盖 60+ 种标签，从 -1.0（崩溃/恐惧）到 +1.0（幸福/兴奋），多标签取均值

## 心情日历

### 长什么样

按月展示，每天是一个**同心圆环**，7 列排列（周日 ~ 周六）。

每个圆环包含三层信息：

| 看什么 | 表达什么 | 数据来源 |
|--------|---------|---------|
| **外环颜色** | 当天的主要情绪 | 第一条事件的 `emotion_tag`，经 `getEmotionColor()` 映射 |
| **内圆明暗** | 能量高低（亮 = 能量足，暗 = 能量低） | `energy` 值 1~10 → `brightness = 0.3 + (energy-1)/9 * 0.7` |
| **环的粗细** | 当天写了多少事（粗 = 事件多） | `density` → `ringWidth = 2.5 + min(density, 4)/4 * 3`（2.5~5.5px） |

「密度」的计算规则（后端 terrain_service）——事件优先，日记兜底：

| 场景 | 密度值 | 说明 |
|------|--------|------|
| 写了 1 篇日记，AI 已分析 | 1 | 1 篇日记对应 1 条事件 |
| 写了 1 篇日记，AI 提炼出 3 个事件 | 3 | 按事件条数 |
| 写了 3 篇日记，都还没分析 | 3 | 没有事件时用日记篇数 |
| 当天什么都没写 | 0 | 显示空心圆环 |

没有日记的日子显示为一个淡灰色幽灵圆（`#E8E6E2`，opacity 0.18），今天用虚线外环标识（`#b56f61`）。

日历底部有一行小图例，解释颜色/明暗/粗细/今天标记的含义。

### 情绪色谱

每种情绪有独立颜色，覆盖 60+ 种情绪标签，按色温分组：

| 色系 | 代表情绪 |
|------|---------|
| 暖阳系（琥珀/暖橘） | 开心、快乐、兴奋、幸福 |
| 柔绿系 | 成就感、成长、自豪、充实 |
| 暖棕系 | 感恩、感动、温暖 |
| 薄荷系 | 平静、放松、释然、满足 |
| 薰衣草系 | 期待、好奇 |
| 雾蓝系 | 疲惫、无聊 |
| 灰紫系 | 困惑、纠结、迷茫 |
| 暗橘系 | 担忧、焦虑、紧张、压力 |
| 青灰系 | 失落、难过、悲伤、孤独 |
| 赤陶系 | 愤怒、崩溃、恐惧 |
| 中性灰 | 平淡、一般 |

> 色谱定义在 `EmotionChart.tsx` 的 `EMOTION_COLORS` 中，全站共用，包含模糊匹配兜底。

### 月份切换

- 左右箭头切换上一月 / 下一月
- 默认显示当月

### 交互

**悬停**（仅当月内有数据的日期）：
- 弹出浮窗，定位在圆环上方，内容由后端 AI 生成：
  > *"03-22 · 平静"*
  > *"这天你去爬山了，感觉很舒畅。"*
- 浮窗数据按需加载（hover 时调用 `daily-insight` API），前端 `dailyInsightMap` 缓存，同一天不重复请求
- 加载中显示"映记精灵正在整理你的记忆..."

**点击**：
- 如果当天有日记（第一条事件有 `diary_id`）→ 跳转日记详情
- 如果只有事件无日记 → 选中该天，更新底部「当日详情」卡片

## 能量值趋势图

### 长什么样

一条**平滑曲线面积图**（Recharts `AreaChart` + `monotone` 插值），X 轴为日期，Y 轴为能量值（1~10 分）。

曲线下方有三段渐变色填充：`#F5C842`（琥珀金，高能量）→ `#E9A6A0`（暖粉，中间）→ `#4A6FA5`（藏青蓝，低能量）。

能量值对应的节点颜色：>= 7 分琥珀金、<= 3 分藏青蓝、其余暖粉色。

### 时间范围

提供 **7 天 / 30 天 / 90 天** 三个 Tab 切换，默认 30 天。切换时重新请求 terrain API。

### 关键事件标注

当某天的能量值相比前一天**变化 >= 2 分**且当天有事件时，在曲线节点上方显示一个小图标：

- 💼 work（工作相关）
- 🤝 relationship（人际关系）
- 🌙 health（健康 / 作息）
- 🏆 achievement（成就）
- ✨ other（其他 / 默认）

点击图标 → 弹出小卡片，显示事件摘要 + 「查看日记 →」按钮。

### 交互

点击曲线上的任意节点 → 页面底部「当日详情」卡片更新为该天的信息。

## 萨提亚关键词卡片

### 设计思路

不用传统词云（太杂乱），而是按照**萨提亚冰山模型**的 5 个层级，分别展示你日记中的高频关键词。

### 布局

横向排列 5 张卡片（`grid-cols-5`），移动端变为纵向（`grid-cols-1`）：

| 卡片 | 标题 | 底色 | 文字色 |
|------|------|------|--------|
| 行为层 | 我做了什么 | `#FFF5E6` 暖橙 | `#D4820A` |
| 情绪层 | 我感受到 | `#F0EBF8` 淡紫 | `#7B5EA7` |
| 认知层 | 我在想 | `#E8F4F0` 薄荷绿 | `#2E7D62` |
| 信念层 | 我相信 | `#EBF2FA` 浅蓝 | `#2563AB` |
| 渴望层 | 我渴望 | `#FEF3F2` 浅粉 | `#C0392B` |

每张卡片内显示最多 8 个关键词，字体大小与权重成正比（`12 + weight*10` px），权重越高越大。

### 数据来源

关键词提取在**前端**完成：将当前 terrain 窗口内所有事件的摘要 + 情绪标签拼为文本，用预定义的层级词表（每层 10 个种子词）做**前端词频匹配**，按出现次数排序取 top 8。

种子词举例：
- **行为层**：做、完成、推进、开发、工作、学习、运动、整理、沟通、安排
- **情绪层**：开心、平静、焦虑、担忧、失落、疲惫、满足、困惑、难过、轻松
- **认知层**：觉得、意识到、反思、想到、理解、判断、计划、逻辑、选择、原因
- **信念层**：应该、必须、价值、意义、相信、原则、重要、成长、坚持、标准
- **渴望层**：希望、想要、渴望、期待、被看见、自由、平衡、陪伴、突破、安全感

> 当事件文本中无命中词时，该层显示空态提示（如"写更多日记，解锁行为洞察"）。

## 当日详情

页面底部有一个「当日详情」卡片，展示：

- 日期（含星期）
- 能量值、愉悦值、事件数（三个标签徽章）
- 当天所有事件的摘要列表，每条包含：摘要文本、情绪标签、重要性评分（x/10）、「查看日记 →」按钮

初始选中最近一天有事件的日期。通过点击日历圆环或趋势图节点来切换查看不同日期。

## 数据自动更新

保存日记时**不做任何分析**，成长中心的数据统一由后端**每日定时任务**（午夜 0:00）生成：

1. 扫描昨天和今天所有有日记记录的用户
2. 对尚未分析的日记，先用规则提取基础事件，再调用大模型精炼摘要、情绪标签、重要性评分
3. 结果写入 `TimelineEvent` 表，下次打开成长中心即可看到

> 唯一的例外：心情日历中 hover 某天时，会按需调用大模型生成当日洞察（`GrowthDailyInsight`），生成后缓存到数据库的 `growth_daily_insights` 表，不重复请求。

### 当日洞察生成逻辑

后端 `/growth/daily-insight` 端点的处理流程：

1. 先查缓存——如果数据库中已有该日期的洞察，直接返回
2. 查询该日的 TimelineEvent + Diary 记录
3. 猜测主要情绪：优先从事件的 `emotion_tag` 取；无事件时从日记的 `emotion_tags` 取
4. 构建回退摘要文本（事件摘要或日记标题/内容前 50 字）
5. 调用 AI 大模型生成一句简短自然的当日总结（30 字以内）
6. 结果写入缓存表后返回

## 交互联动

- 心情日历点击某天 → 跳转日记详情（有日记时）或更新底部当日详情
- 趋势图点击事件标注 → 弹出事件摘要 → 可跳转日记
- 趋势图点击节点 → 底部当日详情更新
- 时间窗口切换（7/30/90 天）→ 趋势图 + 萨提亚关键词同步更新
- 映记精灵在此页面可感知展示的数据，支持提问："我最近状态怎么样？"

## 边界情况

| 情况 | 处理 |
|------|------|
| terrain 数据点少于 3 个 | 显示引导卡片："再写几篇日记，成长洞察将逐渐清晰" + 「去写日记」按钮 |
| 当月没有日记 | 日历全部显示淡灰占位圆 |
| 能量值没有大波动 | 趋势图不显示事件标注，只展示平滑曲线 |
| hover 当月外的日期或无数据日期 | 不触发浮窗 |
| daily-insight 请求失败 | 浮窗显示"获取失败"，不阻塞页面 |
