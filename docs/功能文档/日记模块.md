# 日记模块

> 映记的核心——写日记、看日记、管理日记。

---

## 模块定位

日记模块是映记最基础也最重要的功能，所有的 AI 分析、成长中心、社区内容都建立在日记数据之上。

---

## 路由

| 页面 | 路由 | 说明 |
|------|------|------|
| 日记列表 | `/diaries` | 查看所有日记，支持搜索和过滤 |
| 日记详情 | `/diaries/:id` | 查看单篇日记完整内容 |
| 新建日记 | `/diaries/new` | 写一篇新日记 |
| 编辑日记 | `/diaries/:id/edit` | 修改已有日记 |

---

## 日记包含哪些信息

| 字段 | 说明 |
|------|------|
| 标题 | 可选，不填时列表显示内容前几个字 |
| 内容 | 正文，必填；支持富文本格式（见下方编辑器说明） |
| 日记日期 | 默认今天，可以改成其他日期（补写昨天的日记） |
| 情绪标签 | 多选，如「开心」「焦虑」「平静」等 |
| 重要性评分 | 1~10 分，默认 5，影响成长中心的能量值 |
| 图片 | 支持上传多张图片 |

---

## 日记列表

- 分页展示，每页 20 条
- 支持按**日期范围**和**情绪标签**过滤
- 每条日记显示标题/内容摘要、日期、情绪标签

---

## 写日记 / 编辑日记

编辑器基于 **Lexical** 构建，提供类似 Notion 的富文本编辑体验。

### 富文本编辑器能力

#### 基础格式

- **加粗**、*斜体*、下划线、~~删除线~~、行内代码
- 支持 Markdown 快捷语法（输入 `#` 自动转标题、`>` 转引用、`-` 转列表等）
- 支持标题（H1~H6）、引用、代码块、有序 / 无序列表

#### Notion 风格文字颜色与背景色

选中文字后弹出**浮动工具栏**，支持：

- **文字颜色**：默认、灰、棕、橙、黄、绿、蓝、紫、粉、红（共 10 种，色板参考 Notion）
- **背景高亮色**：无、灰、棕、橙、黄、绿、蓝、紫、粉、红（共 10 种柔和底色）
- **字号调节**：小（13px）、正常（15px）、大（18px）、特大（22px）

颜色和字号通过 Lexical 的 `TextNode.setStyle()` 以内联 CSS 方式存储，前端渲染时即时生效。

#### Notion 风格浮动工具栏

- 选中文字时在选区正下方弹出深色半透明工具栏
- 工具栏包含：拖拽手柄 | 字号 | 文字颜色/背景色 | 加粗 | 斜体 | 下划线 | 删除线 | 行内代码
- 工具栏可**拖拽移动**到屏幕任意位置
- 取消选区后自动收起

#### Slash 命令

- 在编辑区输入 `/` 弹出快捷菜单
- 目前支持：插入图片
- 按 `Enter` 执行、`Esc` 关闭

### 图片功能

- **顶部按钮**：点击工具栏的图片图标选择文件上传
- **Slash 命令**：输入 `/` → 选择「插入图片」
- **Ctrl+V 粘贴**：从剪贴板直接粘贴截图或复制的图片
- 上传到后端 `uploads/diary_images/` 目录
- 文件名自动生成 UUID，防止冲突
- 支持格式：jpg、png、gif、webp

### 数据存储

- 编辑器内容以 **Markdown** 格式存入数据库 `content` 字段
- 图片以 `![alt](url)` 格式内嵌在 Markdown 中
- 同时生成 `content_html` 用于前端渲染
- 保存时只写入 Diary 表，不触发任何 AI 分析（分析集中在每日 0 点定时任务处理）

### 技术实现

| 组件 | 说明 |
|------|------|
| `RichTextEditor.tsx` | 主编辑器组件，基于 `@lexical/react` |
| `ImageNode.tsx` | 自定义 Lexical 图片节点 |
| `FloatingToolbarPlugin` | Notion 风格浮动格式工具栏（含颜色面板） |
| `SlashCommandPlugin` | `/` 快捷命令插件 |
| `PasteImagePlugin` | Ctrl+V 粘贴图片插件 |
| `TopToolbarPlugin` | 编辑区顶部简版工具栏 |

## 删除日记

- 支持删除单篇日记
- 删除后相关的时间轴事件也会清除

---

## 图片上传

- 上传到后端 `uploads/diary_images/` 目录
- 文件名自动生成 UUID，防止冲突
- 返回可访问的 URL 路径
