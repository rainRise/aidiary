# AI 综合分析

> 潜入你的冰山之下——让 AI 帮你回顾一段时间的日记，用萨提亚冰山模型给出深度洞察。

---

## 模块定位

AI 分析模块提供多种深度的日记分析能力，核心是**按需触发**的冰山综合分析。用户主动点击「开始分析」后，后端通过 **RAG 证据收集 + 多智能体萨提亚冰山分析**，从时间窗口内的日记中提炼出行为模式、情绪趋势、深层信念和内心渴望，最终以**冰山卡片**的形式呈现。

与成长中心的区别：成长中心是**被动展示**（打开就能看），综合分析是**主动请求**（需要等待生成）。

---

## 三条分析路径

| | 冰山综合分析 | 用户级整合分析 | 轻量日常分析 |
|---|---------|---------|---------|
| **触发** | 用户在 `/analysis` 页面主动发起 | 用户在 `/analysis/:id` 单篇分析页主动发起 | 每日 0 点定时任务自动执行 |
| **范围** | 时间窗口内的多篇日记 | 锚点日记 + 窗口内日记 + Qdrant 记忆 | 仅当天的日记 |
| **深度** | RAG 证据 + 多智能体萨提亚冰山（5 层 + 疗愈信） | 多智能体萨提亚（5 层）+ 时间轴事件 + 朋友圈文案 | 轻量：提取情绪标签 + 事件 + 简短回应 |
| **端点** | `POST /api/v1/ai/comprehensive-analysis` | `POST /api/v1/ai/analyze` | 后台定时任务 |
| **呈现** | 冰山卡片 + 致你的一封信 | 单篇结果页（时间轴事件 + 萨提亚卡片 + 疗愈回复 + 朋友圈文案） | 数据写入成长中心，不单独展示 |
| **缓存** | 不持久化（每次重新生成） | 结果写入 `AIAnalysis` 表，下次可直接读取 | 结果写入 `TimelineEvent` 表 |

## 冰山综合分析（主路径）

### 使用流程

1. 用户选择分析的**时间窗口**——前端提供 **30 天 / 90 天 / 180 天** 三个选项，默认 90 天
2. 点击「开启冰山之旅」
3. 页面展示冰山引导插画 + 加载动画，提示预计耗时
4. 等待后端多轮大模型调用（通常 20~50 秒）
5. 页面以冰山卡片形式展示分析报告

路由：`/analysis`（需登录）

### 后端分析流程

前端调用 `POST /api/v1/ai/comprehensive-analysis`，后端依次执行三个阶段：

#### Phase 1：RAG 证据收集

```
1. 从数据库读取时间窗口内的所有日记（最多 120 篇）
2. 用 RAG 服务将日记切片 → 生成摘要块 + 原文块
3. 按 5 个维度分别检索相关片段：
   ├── 情绪趋势（焦虑、平静、开心、低落...）
   ├── 连续事件（重复出现的困扰/问题）
   ├── 关键转折（决定、变化、契机）
   ├── 成长线索（进步、反思、改变）
   └── 关系主题（家人、同事、朋友...）
4. 去重后保留最多 18 条证据片段
5. 若所有维度都未命中，回退为"最近的重要事件和情绪变化"宽泛检索
```

#### RAG 检索细节

使用 `rag_service.py` 实现的本地轻量 RAG，不依赖外部向量数据库：

- **切片**：日记按句子边界分段（最大 260 字，40 字重叠）
- **检索**：BM25 关键词匹配 + 多维加权排序
- **排序公式**：`55% 文本相关性 + 14% 时间新近度 + 9% 重要性 + 8% 情绪强度 + 8% 主题重复度 + 4% 人物命中 + 2% 来源加成`
- **去重**：Jaccard 相似度 >= 72% 的片段去重，每篇日记最多取 2 条，每个维度最多 3 条

#### Phase 2：多智能体萨提亚冰山分析

由编排器 `AgentOrchestrator.analyze_iceberg()` 协调，基于 Phase 1 的证据片段，分 6 步逐层调用大模型：

```
Step A → 行为模式识别
         输入: 用户名 + 时间窗口 + RAG 证据片段
         输出: 跨日记的重复行为聚类（如"总是深夜写日记""回避社交"）

Step B → 情绪层分析
         输入: 证据片段 + Step A 结果
         输出: 情绪趋势描述 + 转折点识别 + 情绪色块数据（emotion_flow）

Step C → 认知层分析
         输入: 证据片段 + Step A/B 结果
         输出: 反复出现的思维模式 / 自动化想法（如"觉得自己不够好"）

Step D → 信念层分析
         输入: 证据片段 + Step A/B/C 结果
         输出: 深层核心信念 + 自我叙事主线

Step E → 渴望层分析
         输入: Step A~D 全部结果 + 证据片段
         输出: 内心需求描述 + 生命能量

Step F → 致你的一封信
         输入: 五层完整分析结果
         输出: 温暖的疗愈信（纯文本，非 JSON）
```

每个步骤独立调用大模型（temperature 0.4，JSON 模式），疗愈信用 temperature 0.8 以增加温度。每步耗时记录在 `agent_runs` 中。

#### Phase 3：结构化报告输出

将五层分析结果 + 疗愈信 + 证据引用 + 元数据（窗口、日记数、证据数、耗时、各步耗时）组装为 `IcebergAnalysisResponse`。

### 视觉设计要点

- **渐变色**：卡片背景从上到下——浅天蓝 → 蓝 → 靛蓝 → 深紫 → 暗紫金，页面整体背景也从浅蓝渐变到深靛夜空，模拟从水面潜入深海
- **冰山 hero 装饰图**：分析前显示冰山插画（`/images/iceberg-hero_1_no_bg.png`），分析中显示引导插画（`/images/iceberg-guide_1.png`）
- **展开/折叠**：每层卡片右上角「展开 / 收起」按钮，展开后显示该层的详细子项（行为模式列表、情绪阶段、思维模式触发、核心信念来源、渴望列表等）
- **情绪色块条**：情绪层卡片中嵌入一个横向色块条，暖色（`#fb923c`）/ 冷色（`#60a5fa`）/ 中性（`#a1a1aa`）表达情绪走势
- **滚动体验**：整页向下滚动 = 向内心深处下潜，每张卡片进入视口时有淡入 + 上滑动画（延迟递增 120ms）
- **致你的一封信**：最底部的疗愈信，使用「霞鹜文楷 / 楷体」手写风格字体 + 淡黄信纸渐变背景 + 半透明信纸纹理叠加
- **深浅层差异**：前三层（行为/情绪/认知）为浅色系文字；后两层（信念/渴望）切换为白色系文字以适配深色背景

## 单篇分析（用户级整合路径）

路由：`/analysis/:id`（需登录，`:id` 为日记 ID）

用户在日记详情页点击「AI 分析」按钮进入。后端 `POST /api/v1/ai/analyze` 以该篇日记为锚点，向前扩展时间窗口聚合多篇日记进行整合分析。

### 分析内容

```
1. 聚合锚点日记 + 窗口内多篇日记为用户级语料
2. 从 Qdrant 向量库检索相关历史记忆（最多 4 条，未配置时跳过）
3. 获取时间轴历史上下文
4. 多智能体分析：
   ├── 上下文收集 → 时间轴事件提取
   ├── 萨提亚冰山 4 层（情绪/认知与信念/存在层/疗愈回复）
   └── 朋友圈文案生成（3 条 ABC 版本）
5. 结果持久化到 AIAnalysis 表 + TimelineEvent 表
```

### 页面呈现

```
┌──────────────────────────────────────────┐
│  ← 返回日记  首页     ✨ AI分析          │
│                                          │
│  ┌─ 日记摘要 ────────────────────────────┐
│  │  标题 · 日期 · 情绪标签              │ │
│  │  内容预览（最多 3 行）               │ │
│  └───────────────────────────────────────┘
│                                          │
│  ┌─ 时间轴事件 ──────────────────────────┐
│  │  📅 事件摘要 + 情绪标签 + 重要性     │ │
│  └───────────────────────────────────────┘
│                                          │
│  ┌─ 萨提亚冰山分析 ─────────────────────┐
│  │  ❄ 情绪层：表层情绪 / 深层情绪      │ │
│  │  ⚡ 认知层：非理性信念列表           │ │

│  │  🌹 存在层：最深渴望                 │ │

│                                          │
```

## 轻量日常分析

每日 0 点定时任务自动处理当天和昨天的日记，不走完整萨提亚：

```
1. 提取情绪标签 + 重要性评分
2. 提取时间轴事件（结构化）
3. 生成简短温暖回应
```

结果写入数据库，供成长中心展示。不单独生成页面。

## 辅助 API

除核心分析端点外，AI 模块还提供以下辅助功能：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/ai/generate-title` | POST | 根据日记内容生成简洁标题（10 字以内） |
| `/api/v1/ai/daily-guidance` | GET | 基于近 30 天日记上下文生成每日个性化引导问题 |
| `/api/v1/ai/social-posts` | POST | 单独生成朋友圈文案（3 条 ABC 版本），支持风格样本学习 |
| `/api/v1/ai/social-style-samples` | GET/PUT | 获取/上传朋友圈风格样本（最多 50 条） |
| `/api/v1/ai/satir-analysis` | POST | 仅萨提亚分析（简化版，返回占位结构） |
| `/api/v1/ai/analyses` | GET | 获取用户历史分析记录列表 |
| `/api/v1/ai/result/{diary_id}` | GET | 获取指定日记的最近一次已缓存分析结果 |
| `/api/v1/ai/analyze-async` | POST | 异步分析（当前实现等同同步，预留 Celery 扩展） |
| `/api/v1/ai/models` | GET | 获取可用 AI 模型及多智能体系统信息 |

## 记忆与数据积累

| 使用场景 | 是否使用历史日记 | 记忆方式 |
|----------|----------------|---------|
| 冰山综合分析 `/analysis` | Yes 读取时间窗口内的全部日记 | 本地 RAG（BM25 检索）-> 多智能体分析 |
| 单篇整合分析 `/analysis/:id` | Yes 锚点日记 + 窗口内日记 | Qdrant 向量检索（如已配置）+ 多智能体分析 |
| 映记精灵对话 | Yes 检索相关日记片段 | Qdrant 向量检索（如已配置）或本地 RAG 兜底 |
| 轻量日常分析（定时任务） | No 只处理当天日记 | 轻量提取，不走 RAG |

> Qdrant 需要在 `.env` 中配置 `QDRANT_URL` 和 `QDRANT_API_KEY` 才会启用，未配置时 Qdrant 检索静默跳过、降级为无记忆模式。

## 交互细节

- 分析前页面展示冰山 hero 装饰图 + 时间窗口选择器
- 点击「开启冰山之旅」后显示引导插画 + 加载动画 + "正在检索你的日记…" 提示，按钮置灰不可重复触发
- 分析完成后五层卡片依次淡入（延迟 120ms 递增），「下潜指示器」连接各层
- 每层卡片可展开查看详细子项（行为模式、情绪阶段、思维模式、核心信念、渴望列表等）
- 分析失败时显示红色错误提示，可重试
- 页面底部显示元数据：耗时、证据条数
- 冰山综合分析结果不持久化，刷新页面后需重新分析；单篇分析结果持久化到数据库
- 左上角返回按钮 -> 回到首页
