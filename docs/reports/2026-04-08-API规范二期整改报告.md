# 2026-04-08 API规范二期整改报告

> 范围：REST 风格统一（兼容迁移）、成功响应统一包裹、前端联动解包、`/api/doc` 文档增强

---

## 一、整改目标与结论

本次按照规则文件的三项要求完成了可上线版本：

1. 历史动作式 URL：新增 REST 风格别名，保留旧路径兼容。  
2. 成功响应统一：后端对 `/api/v1/*` 成功 JSON 响应统一包裹 `{code,message,data,request_id}`。  
3. 前端联动：在 `frontend/src/services/api.ts` 统一解包 `response.data = payload.data`，避免页面层逐页改造。

---

## 二、已完成代码改造

## 1) 成功响应统一包裹
- 文件：`backend/main.py`
- 方案：`SuccessEnvelopeMiddleware`
- 规则：
  - 仅处理 `/api/v1/*`
  - 仅处理成功响应（`<400`）与 `application/json`
  - 跳过流式接口：`/api/v1/assistant/chat/stream`
  - 已是规范结构时不重复包裹

包裹后统一形态：

```json
{
  "code": 200,
  "message": "Success",
  "data": { "...": "..." },
  "request_id": "uuid"
}
```

## 2) REST 风格别名（兼容迁移）

### 认证模块（`backend/app/api/v1/auth.py`）
- `POST /api/v1/auth/registration-codes` → 兼容 `register/send-code`
- `POST /api/v1/auth/registration-verifications` → 兼容 `register/verify`
- `POST /api/v1/auth/login-codes` → 兼容 `login/send-code`
- `POST /api/v1/auth/password-sessions` → 兼容 `login/password`
- `POST /api/v1/auth/verification-sessions` → 兼容 `login`
- `POST /api/v1/auth/password-reset-codes` → 兼容 `reset-password/send-code`
- `POST /api/v1/auth/password-resets` → 兼容 `reset-password`
- `POST /api/v1/auth/token-refreshes` → 兼容 `refresh`

### 日记模块（`backend/app/api/v1/diaries.py`）
- `POST /api/v1/diaries/diary-images` → 兼容 `upload-image`
- `POST /api/v1/diaries/timeline-events/rebuild` → 兼容 `timeline/rebuild`

### 社区模块（`backend/app/api/v1/community.py`）
- `POST /api/v1/community/community-images` → 兼容 `upload-image`
- `POST /api/v1/community/posts/{post_id}/likes` → 兼容 `like`
- `POST /api/v1/community/posts/{post_id}/collections` → 兼容 `collect`

### 外部接入（`backend/app/api/v1/integrations.py`）
- `POST /api/v1/integrations/openclaw-tokens` → 兼容 `openclaw/token`
- `DELETE /api/v1/integrations/openclaw-tokens` → 兼容 `openclaw/token`
- `POST /api/v1/integrations/openclaw-entries` → 兼容 `openclaw/ingest`

## 3) 前端联动解包
- 文件：`frontend/src/services/api.ts`
- 方案：在 Axios 响应拦截器中自动识别 `{code,message,data,request_id}` 并解包为 `data`
- 结果：业务 service 层与页面层无需逐页修改。
